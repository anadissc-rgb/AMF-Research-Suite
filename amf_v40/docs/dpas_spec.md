# DPAS Formal Specification

**Discrete Positional Alignment System — AMF v4.0**

Status: **PROVISIONAL HYPOTHESIS** — not linguistically proven.
Version: 4.0.0-alpha

---

## Overview

The DPAS is a formal constraint model over Voynichese token sequences.
It proposes that Voynichese tokens are generated by a finite-slot template
with defined character-class constraints at each position.

This document specifies the model formally. It should be read alongside
`amf/dpas/constraints.py` (the implementation) and `docs/limitations.md`
(the known limitations).

---

## Formal Definition

### Token Template

A Voynichese token W is modeled as a sequence over five ordered slots:

```
W ::= PREFIX? GALLOWS? CORE SUFFIX?
```

Where:
- `?` indicates an optional (zero or one) occurrence
- `CORE` is required (every well-formed token must have a core element)

### Slot Definitions

Let Σ be the EVA character vocabulary including multigraphs.

**PREFIX** ∈ {d, s, l, q, o}
- Optional
- Occurs at most once
- Hypothesis: may function as determiner or morphological prefix

**GALLOWS** ∈ {k, t, p, f, ck, ct, cp, cf, sk, st, sp, sf}
- Optional
- Occurs at most once
- Hypothesis: structurally prominent, may carry semantic weight

**CORE** ∈ {o, a, e, i, ch, sh, ol, or, al, ar, ee, eee, ii, iii,
              ain, aiin, aiiin}
- Required
- Occurs one or two times (reduplication attested in corpus)
- The "ain/aiin/aiiin" series is a recurring unit of unknown function

**SUFFIX** ∈ {y, m, n, r, l, in, an, on, s}
- Optional
- Occurs at most once
- Hypothesis: may function as grammatical suffix or positional marker

## Coverage Metric

The primary falsifiability criterion of DPAS is **coverage**:

```
coverage = |{w ∈ Corpus : valid(w)}| / |Corpus|
```

Where `valid(w)` is True iff the token `w` can be fully parsed by the
template above (no unconsumed characters after template exhaustion,
no required slots unsatisfied).

**Current threshold**: coverage ≥ 0.70 is required to consider the model
supported. This threshold is **provisional** — see `docs/limitations.md §3.3`.

## Limitations of This Specification

1. Slot definitions were derived from the same corpus they will be tested on
   (circularity risk — see `docs/limitations.md §4.3`)
2. Multigraph segmentation is ambiguous; different segmentations yield
   different coverage values
3. The CORE set may be incomplete — rare tokens may have valid structure
   not captured by the current definition
4. No formal proof exists that these slots are unique or minimal
