# AMF v4.0 — Data Directory

This directory holds Voynich Manuscript EVA transcription files and
processed outputs. **No corpus data is committed to this repository**
due to licensing and provenance traceability requirements.

---

## Required: EVA Transcription Files

Place EVA transcription files in `data/eva/` before running the pipeline.

### Recommended Source

**Zandbergen EVA Transcription (interlinear format)**
- URL: https://www.voynich.nu/transcr.html
- Format: `.vtt` interlinear files
- License: CC BY-NC-SA (check current terms on the site)
- Version: Record the exact version/date for reproducibility

### Alternative Sources

**voynich-text (plain-text EVA)**
- URL: https://github.com/jedahan/voynich-text
- Format: plain text, one word per line or space-separated
- License: Public domain

**Takahashi transcription**
- URL: https://www.voynich.nu/transcr.html (listed on same page)
- Different transcriber — useful for inter-transcriber comparison

---

## Reproducibility Requirement

**Always record the transcription version** when running the pipeline:

```bash
python -m amf.corpus.ingest \
    --input data/eva/ \
    --version "Zandbergen-2024-01-15" \
    --output data/processed/corpus.json
```

The version string is stored in corpus metadata and propagates to all
output reports. Without a version string, results cannot be precisely
reproduced.

## Inter-Transcriber Variation

Different EVA transcribers disagree on approximately 5–10% of characters.
This uncertainty propagates to all statistical analyses. AMF v4.0 recommends:

1. Running the full pipeline on at least two different transcriptions
2. Reporting results from both and documenting disagreements
3. Treating results that differ substantially between transcriptions
   as **unreliable** rather than averaging them

This has not yet been implemented in AMF v4.0 alpha — it is a planned
validation step (see `docs/future_work.md`).
